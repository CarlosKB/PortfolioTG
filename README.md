### 2022-1

<Div align="justify" >

> [Link para a API do 1º Semetre](https://github.com/4DeskGroup/API-2022.1)

### Problema
O projeto de Aprendizagem por Projetos Integrados do 1º Semestre do curso de Análise e Desenvolvimento de Sistemas surgiu a partir de uma demanda identificada pelo cliente interno, professor Antônio Egydio São Thiago Graça, da Fatec. O problema abordado foi a dificuldade dos alunos e profissionais em encontrar oportunidades de emprego e cursos de certificação de forma eficiente. Sem uma plataforma centralizada, a busca por essas oportunidades era feita manualmente, o que tornava o processo demorado e ineficiente, exigindo que os usuários acessassem múltiplos sites e fontes para coletar informações relevantes.

### Solução
A solução proposta foi o desenvolvimento de um site web dedicado à catalogação de vagas de emprego e cursos profissionalizantes, utilizando técnicas de raspagem de dados para coletar informações relevantes de diversas fontes. O site oferece uma interface amigável e organizada, permitindo que os usuários busquem e visualizem facilmente as oportunidades disponíveis em um único local. As tecnologias utilizadas, como HTML, CSS e Python, garantiram um site funcional e esteticamente agradável, melhorando a experiência do usuário e facilitando o acesso às informações.

### Problema
No projeto de Aprendizagem por Projetos Integrados do 2º Semestre do curso de Análise e Desenvolvimento de Sistemas, identificou-se uma necessidade crítica da TrackCash, uma empresa do setor financeiro especializada em controle de gastos. A empresa gerenciava e protegia dados sensíveis de seus clientes parceiros, como canais de marketplaces, meios de pagamento, plataformas ERP e credenciais de acesso, incluindo senhas e tokens, de forma desorganizada e descentralizada. Isso dificultava a integração e o acesso eficiente a essas informações, gerando desafios na segurança e na gestão desses dados financeiros sensíveis. 

### Solução
A solução proposta foi o desenvolvimento de um aplicativo desktop para o registro de canais de vendas, permitindo à TrackCash interagir de maneira eficiente com as informações coletadas. O aplicativo foi criado utilizando tecnologias como Java para o desenvolvimento do front-end e back-end, MySQL para gerenciamento de banco de dados e Amazon AWS para a hospedagem do banco de dados. Essa abordagem não só atendeu às necessidades específicas do cliente, mas também proporcionou uma interface amigável e uma lógica de funcionamento robusta, melhorando significativamente a eficiência no controle de vendas.

### Problema
O projeto de Aprendizagem por Projetos Integrados do 3º Semestre do curso de Análise e Desenvolvimento de Sistemas teve como parceira a Visiona, uma empresa do setor espacial. A empresa enfrentava dificuldades na gestão de usuários, já que o controle de acessos e operações no sistema era feito de forma manual e descentralizada. A solução desenvolvida foi um sistema de Gerenciamento de Usuário baseado em tecnologia de microserviços. Utilizando React para o front-end e JavaScript para o back-end, além de PostgreSQL para gerenciamento de banco de dados, o sistema proporciona uma interface amigável e funcionalidades robustas, como login, recuperação de senha e dashboards informativos. A implementação e o deployment foram realizados na plataforma Vercel, garantindo uma solução completa e integrada para a Visiona.

<Div align="justify" >

> [Link para a API do 4º Semetre](https://github.com/4DeskGroup/API-2023.2)

### Problema
O projeto de Aprendizagem por Projetos Integrados do 4º Semestre no curso de Análise e Desenvolvimento de Sistemas foi desenvolvido em parceria com a Greenneat, uma empresa do setor de controle e descarte de óleo usado. A Greenneat enfrentava dificuldades na gestão de usuários e seus créditos, que funcionavam como moeda de troca em transações de óleo usado, permitindo aos clientes obter benefícios e descontos em sua loja virtual. O processo de controle dessas transações, como compras e vendas de créditos, era feito manualmente, o que dificultava a rastreabilidade e o gerenciamento eficiente das informações. Isso causava confusões e problemas de comunicação entre a empresa e seus clientes parceiros.

### Solução
A solução proposta foi o desenvolvimento de uma plataforma que gerencia os créditos de forma centralizada, permitindo que os usuários acompanhem suas contrapartidas durante o processo de coleta. A aplicação foi desenvolvida utilizando React para o front-end, JavaScript para o back-end, e PostgreSQL para o gerenciamento do banco de dados. O sistema inclui funcionalidades como login, recuperação de senha e gerenciamento de créditos, além de uma interface intuitiva que facilita a navegação e a visualização das informações. O deploy da aplicação foi realizado na plataforma Vercel, garantindo uma solução acessível e integrada para a GREENNEAT.
